## המודל — שתי דלתות, מוח אחד

לכל סוכן שלושה קבצים בתיקייה שלו:
- **`AGENT.md`** — המוח: זהות, הידע המחלקתי, החוקים, הפרוטוקולים, הגבולות.
- **`CLAUDE.md`** — הדלת במחשב: פתיחת Claude Code בתיקיית הסוכן = שיחה ישירה איתו.
- **`MEMORY.md`** — הזיכרון החי שלו: נקודות המשך, לקחים, מצב עדכני. הסוכן מעדכן אותו תוך כדי עבודה.

**דלת 1 — במחשב (הדרך המועדפת):** פותחים Claude Code בתוך תיקיית הסוכן ומדברים איתו ישירות.
**דלת 2 — בדרכים:** בוט טלגרם ייעודי דרך שער הסוכנים (agent-gate) — אותו AGENT.md בדיוק.

## עקרון העצמאות (חוקה, עיקרון 1)

כל סוכן חי רק בתיקייה שלו וכותב זיכרון רק לקבצים שלו. סוכנים לא תלויים זה בזה — הסנכרון היחיד הוא דרך המאנדיי. קריסת סוכן אחד לא נוגעת לשאר.

## פריסה במחשב (סשן מקומי — פעם אחת)

1. `git pull` של הריפו.
2. לכל סוכן חדש: יצירת בוט ב-BotFather ← הוספה ל-`C:\dev\agent-gate` (טוקן + prompts) ← ה-prompt מפנה ל-`AGENT.md` של הסוכן בריפו.
3. בדיקת חיים: הודעה אחת לכל בוט.

הסוכנים הוותיקים (המסכם, שומר זמן, סבג-מאנדיי) אינם כאן — הם שייכים לפרויקטים Call-Recording, Smart-Time ו-Monday-Operations.
