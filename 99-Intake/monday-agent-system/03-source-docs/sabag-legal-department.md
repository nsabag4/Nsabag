# מחלקה משפטית — התהליך המלא: מלקוח נכנס ועד תיק סגור — סבג

*בנוי על מאנדיי. אוטומציות ממוספרות L1–L15 — מגדירים אחת-אחת. משלים את מסמך הסוכן התפעולי.*

## עקרונות העל

1. **אין אירוע בלי תאריך. אין תאריך בלי תזכורת. אין תזכורת בלי אחראי.** זו כל השיטה.
2. **יומן תיק:** כל מה שקרה בתיק רשום כרונולוגית — בתאריך X קרה כך וכך. לקוח שואל "מה קורה עם התיק?" → תשובה מדויקת תוך 30 שניות, מהיומן.
3. **לקוח מעודכן יזום — לא לקוח ששואל.** הלקוח שומע מאיתנו לפני שהוא מרים טלפון.
4. **הכסף הוא חלק מהתיק.** אבני חיוב מוגדרות בפתיחה, חשבונית יוצאת בו-ביום שהשלב הושלם, גבייה רצה אוטומטית.

## מבנה התיק בלוח המשפטי

- **הפריט = התיק.** קבוצות בלוח לפי מסלול: חוזים | הסכמי מכר | מו"מ מסחרי | ליטיגציה.
- **תתי-משימות = שלבי הצ'קליסט של המסלול** + מסמכים חסרים (כל מסמך חסר = תת-משימה עם אחראי: אנחנו / הלקוח / צד שלישי).
- **עמודות:** לקוח (Connect ל-CRM) | שלב (סטטוס) | אחראי | האירוע הבא | תאריך האירוע הבא | ממתינים לצד שני עד | שכ"ט ואבן חיוב הבאה | סטטוס חיוב | VIP | ערכאה ומספר הליך (לליטיגציה).
- **יומן התיק = העדכונים (Updates) על הפריט**, בפורמט קבוע: [תאריך] מה קרה → מה זה אומר → הצעד הבא ומתי.

## קליטת תיק — מהרגע שהלקוח סגר ונתן מקדמה

1. פתיחת פריט בקבוצת המסלול + קישור לכרטיס הלקוח ב-CRM (לקוח 360 → מקושר גם למטה ה-VIP).
2. **L1 — תיק חדש נפתח:** טעינת תבנית המסלול כתתי-משימות, עם תאריכי יעד לפי משכי השלבים.
3. **L2 — תיק חדש נפתח:** משימה להיום — הסכם שכ"ט חתום + מכתב פתיחת תיק ללקוח: מה התהליך, השלבים, לו"ז צפוי, ורשימת המסמכים שנדרשים ממנו. **אין עבודה על תיק בלי הסכם שכ"ט חתום.**
4. אבני החיוב נקבעות בפתיחה ונרשמות בעמודה.

## ניהול הזמן — שלושה סוגי תאריכים

**תאריך פנימי (שלנו):** "טיוטה עד יום חמישי."
- **L3:** תזכורת לאחראי יומיים לפני.

**תאריך של הצד השני:** "הם צריכים להשיב עד ה-15."
- נרשם בעמודת "ממתינים לצד שני עד". התיק בהמתנה — אבל לא נשכח.
- **L4 — הגיע התאריך ואין תשובה:** משימת נדנוד אוטומטית לאחראי + רישום ביומן + הודעה ללקוח שאנחנו רודפים אחרי הצד השני.

**תאריך נעול (דיוני / סטטוטורי):** דיון, הגשת כתב טענות, דיווח מיסוי. אי אפשר להזיז.
- **L5:** התראות מדורגות — 14 ימים לפני, 7, 3, ויום לפני.

## המסלולים

### חוזים

קבלת חומרים (יעד: יומיים; חסר → תתי-משימות מסמכים) → טיוטה ראשונה (5 ימי עבודה) → נשלח לצד השני + "ממתינים עד" → סבבי תיקונים (כל סבב: 3 ימים) → חתימה (תיאום מעמד) → תיוק + מכתב סיכום ללקוח.

- **L12 — נפתח סבב תיקונים רביעי:** דגל לנחי — שיחה עם הלקוח על היקף ושכ"ט לפני שממשיכים.
- חיוב מוצע: 50% בפתיחה, 50% בחתימה (או לפי הסכם שכ"ט).

### הסכמי מכר

בדיקת זכויות ונסח (3 ימים) → טיוטה (5 ימים) → מו"מ ("ממתינים לצד שני") → חתימה → דיווח מיסוי → תשלומי מיסים לפי שוברים → רישום בטאבו → סגירה במכתב סיכום + מסירת כל המסמכים ללקוח.

- **L8 — סומנה חתימה:** נוצר אוטומטית תאריך נעול לדיווח המיסוי, מחושב מיום החתימה לפי המועד הקבוע בדין (נחי מגדיר את מספר הימים בתבנית פעם אחת), כולל ההתראות המדורגות של L5.
- חיוב מוצע לפי אבני דרך: חתימה / דיווח / רישום.

### משא ומתן מסחרי

מיפוי אינטרסים (פגישת פתיחה + מסמך יעדים מתועד ביומן) → הצעה → סבבי מו"מ (כל סבב: רישום ביומן + "ממתינים עד") → סיכום עקרונות → מעבר שלב לחוזה — אותו תיק ממשיך למסלול חוזים.

### ליטיגציה — הקשוח מכולם

1. כל תיק: ערכאה + מספר הליך בעמודות.
2. **כל מועד דיוני = תת-משימה עם תאריך נעול:** דיון, כתב טענות, תגובה, תצהירים. **אין מועד דיוני בלי פריט.**
3. **L6 — נקבע מועד דיון:** נוצרת אוטומטית משימת "הכנה לדיון" שבוע לפני, עם צ'קליסט: תיק מוצגים מלא? עדים תואמו? קו טיעון? מסמכים חסרים 1, 2, 3 → כל אחד תת-משימה עם אחראי ותאריך.
4. אחרי כל דיון — בו-ביום: עדכון יומן (מה נקבע), ורישום המועד הבא.
5. **L7 — עבר מועד דיון בלי סיכום ביומן ובלי מועד הבא:** דגל אדום לנחי בו-ביום. דיון שעבר בלי תיעוד = התיק הכי מסוכן במשרד.
6. ספי קיפאון מקוצרים: 3 ימים ללא תנועה → אחראי; 7 → נחי.

## יומן התיק וחוויית הלקוח

1. **L9 — כל מעבר שלב:** רישום אוטומטי ביומן + תזכורת "לעדכן את הלקוח" (שדרוג אפשרי: אימייל אוטומטי בתבנית).
2. נקודות עדכון קבועות ללקוח: פתיחה | כל מעבר שלב מהותי | לפני ואחרי דיון | חתימה | סגירה.
3. **L15 — לקוח VIP (מסלול 360):** עדכון יזום אחת לשבועיים גם כשאין אירוע — "התיק בשלב X, הצעד הבא Y בתאריך Z."
4. פניית לקוח → תשובה תוך 24 שעות לכל היותר (היעד הקבוע). היומן נותן את התשובה מיד.

## שכר טרחה וגבייה

1. הסכם שכ"ט חתום לפני תחילת עבודה (L2) — בלי יוצא מהכלל.
2. **L10 — שלב חיוב הושלם:** משימת "להוציא חשבונית" בו-ביום. החשבונית נרשמת בלוח התזרים אצל סוכן הפנים.
3. **L11 — חשבונית שלא שולמה:** מטריצת הגבייה של החברה — 1–7 ימי איחור: "ללחוץ עכשיו" | 8–30: "מתעכב" | 30+: "אכיפה משפטית". התראה בכל מעבר מדרגה.
4. עבודה מעבר להסכם (סבב רביעי, היקף שגדל) → סימון לחיוב + דגל לנחי להחלטה. לא עובדים בחינם בשקט.
5. **L14 — תיק מסומן לסגירה עם יתרת חוב פתוחה:** נחסם — נדרשת החלטת נחי מתועדת ביומן.

## רשת ביטחון כללית

1. **L13 — תיק ללא תנועה** (ולא בסטטוס "ממתינים לצד שני"): 7 ימים → תזכורת לאחראי; 14 → נחי. בליטיגציה: 3 ו-7.
2. כל תיק, בכל רגע: אחראי + שלב + תאריך אירוע הבא. חסר אחד → תיק יתום → דגל.
3. סריקת הסוכן השבועית: תיקים יתומים, ממתינים לצד שני באיחור, מסמכים חסרים פתוחים, חיובים שלא הוצאו, דיונים ב-14 יום הקרובים.

## דוח שבועי מחלקתי

1. תיקים לפי מסלול ושלב.
2. דיונים ומועדים נעולים ב-14 הימים הקרובים + סטטוס הכנה.
3. ממתינים לצד שני — מי באיחור וכמה זמן.
4. כסף: חיובים שהבשילו ולא הוצאו, חובות פתוחים לפי גיל.

## הקמה — סדר פעולות

1. עמודות ותוויות שלב בלוח המשפטי, לפי המבנה למעלה, כולל קבוצות ארבעת המסלולים.
2. בניית תבניות תתי-המשימות לכל מסלול (הצ'קליסטים) + הגדרת מספר הימים לדיווח מיסוי בתבנית המכר.
3. הקמת האוטומציות לפי הסדר: L1 עד L15.
4. תיק דמה קצה-לקצה: פתיחה → תבנית נטענה? מכתב פתיחה נוצר? תאריך נעול מדומה מפעיל התראות? חיוב מייצר משימת חשבונית? רק כשהכול עובד — המחלקה חיה.
