## מטרה
בנייה של אוטומציות ואינטגרציות במאנדיי עבור מערכת סבג נדל"ן, לפי המספור הקבוע במסמכי הידע (A1–A20 לידים, L1–L15 משפטי, F1–F16 מימון, M1–M12 הצלבות, P1–P16 פרויקטים, B1–B6 לוח הבוקר).

## מתי מפעילים את הכישור
רק בהוראה מפורשת מנחי להיכנס למצב הקמה או שיפור (מצב 1 או 2). לעולם לא באמצע עבודה שוטפת.

## סדר עדיפויות מחייב
1. **קודם — אוטומציה מובנית של מאנדיי** (Automations Center): טריגרים כמו When item created / When status changes / When date arrives / Every time period, ופעולות כמו notify / change status / set date / create item / create task.
2. **אחר כך — אינטגרציה מובנית ממרכז האינטגרציות** של מאנדיי: פייסבוק לידים, Gmail, Outlook, וואטסאפ, Twilio וכו'.
3. **גורם חיצוני (Make/Zapier/API חיצוני) — רק אם אין פתרון מובנה, ורק באישור מפורש מנחי.**
פחות גורמים זרים = פחות נקודות כשל, פחות עלויות, הכול במערכת אחת.

## שיטת עבודה — לכל אוטומציה
1. פתח את מסמך המחלקה וקרא את הגדרת האוטומציה הממוספרת המדויקת (טריגר, תנאי, פעולה, ספים).
2. קרא את מבנה הלוח (get_board_info) — מזהי עמודות ותוויות סטטוס מדויקות. לעולם אל תמציא תווית.
3. בנה את האוטומציה עם התוויות המדויקות מהלוח. אם ההגדרה דורשת יכולת שלא קיימת באוטומציות מובנות (השוואת מספרים בין עמודות, פעולה בין-לוחית מורכבת, ספירת דחיות) — עצור וסמן: "דורש עמודת נוסחה / ריצת סוכן / אישור נחי לפתרון חיצוני". אל תבנה קירוב שגוי בשקט.
4. **בדיקת קצה-לקצה עם פריט דמה** (שמו מתחיל ב"בדיקה"): הפעל את הטריגר בפועל וודא שהפעולה קרתה. רק אחרי שעובד — האוטומציה חיה.
5. דווח לנחי בשורה אחת: מה נבנה, על איזה לוח, מה נבדק.

- אין יצירת לוחות, עמודות או קבוצות בלי אישור מפורש מנחי (חוק ברזל 6). עמודת עזר שנדרשת לאוטומציה (למשל עמודת נוסחה למרווח F12) — מבקשים אישור קודם.
