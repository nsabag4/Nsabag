# מערכת סוכני מאנדיי — סבג נדל"ן | תיעוד מלא של סשן ההקמה

**תאריך:** 05–06.07.2026 | **בעלים:** נחי אביטן (עו"ד, בעלים של סבג נדל"ן) | **מבצע:** קלוד קוד (Claude Fable 5) עם חיבור MCP למאנדיי (חשבון sabag-nadlan, משתמש 43672192)

## 1. מטרת הפרויקט

להקים לסבג נדל"ן מערכת ניהול מלאה על monday.com המופעלת על ידי סוכני בינה מלאכותית, לפי מסמכי תכנון שנחי סיפק (קובץ ZIP עם 8 מסמכים). עקרונות העל:
- **שני מרחבים:** סוכן תפעול (חוץ — לקוחות, לידים, מחלקות) וסוכן פנים (מנכ"ל/CFO — תזרים, כסף, פרוטוקול ADHD).
- **מקסימום אוטומציות ואינטגרציות מובנות של מאנדיי** (חינם ויציב); גורם חיצוני רק באישור.
- **קלוד קוד הוא הסוכן בפועל** (החלטת נחי, 06.07): סוכני ה-AI המובנים של מאנדיי צורכים קרדיטים = כפל תשלום. סוכן מאנדיי 44092 נשאר פעיל רק כשער וואטסאפ.

## 2. מה הוקם בפועל (הכול נבדק קצה-לקצה אלא אם צוין)

### מסמכי ידע במאנדיי (סביבת SABAG 360, doc ids 9308678–9308702)

### עמוד השדרה של הקליטה (הושלם ונבדק 06.07 ערב)
- **לוח אנשי קשר (1494474506):** נוספו עמודות תאריך, הסבר, קישור לתיק, סטטוס איש קשר. עצמאי לחלוטין — בלי חיבור לוחות (החלטת נחי: נכס שיווקי נצחי).
- **לוח "🗂️ תיקי לקוחות | Master" (5099714404):** תיק=לקוח. נוספו: סוג לקוח—נתב (רוכש דירה/משקיע/לקוח מחלקות/לקוח פרויקט), רווחיות—קובעת עדיפות, טלפון, מייל, כרטיס איש קשר (לינק), וואטסאפ (עמודת נוסחה שבונה קישור שיחה מהטלפון).
- **אוטומציית העתקה:** איש קשר חדש ← תיק לקוח (כולל טלפון, מייל, תאריך ולינק חזרה). ✅ נבדק — רץ תוך שניות.
- **נתב חדר המיון (לוח 5094204623):** עמודת סטטוס "ניתוב אל" עם 6 תחנות + 5 אוטומציות ניתוב: פנים←מנהל משימות חכם, משפטי, שיווק ומכירות, מימון, תכנון ובניה — כל אחת יוצרת פריט בלוח המחלקה בקבוצת "לטיפול עכשיו" ומסמנת "בוצע ניתוב". ✅ נתיבי פנים ומשפטי נבדקו בזמן ריצה (3–4 שניות); השאר באותו מנגנון מוכח.
- כל פריטי הבדיקה נמחקו באישור מפורש של נחי.

## 3. מה תוכנן וטרם הוקם

1. **יעד לתווית "ביצוע / פרויקטים"** בנתב — אין לוח משימות לביצוע; ממתין להחלטת נחי (קומנד סנטר / לוח ישן / לוח חדש באישור).
2. **מסלול הלקוח בתוך המחלקות** — הצעד הבא המוסכם: משפטי קודם (ליטיגציה / מכר רגיל / מכר מורכב / כללי + תתי-משימות), אחר כך מימון ותכנון ובניה. אחריו: עמודות רוכש דירה ומשקיע בסי-אר-אם, לקוח פרויקט (זלמן ארן 1 עם כספים+ביצוע).
3. **המרחב הפנימי** — נעול למורשים בלבד; נבנה אחרי החוץ.
4. **ניקוי חדר המיון:** 5 פריטי דמו ישנים, 2 קבוצות "Group Title", עמודת סטטוס כפולה; גורל הלוחות הכפולים (שער הקלט הפיננסי, מעקב מנכ"ל, קומנד סנטר).
5. **הקמת המסלולים מהמסמכים:** האוטומציות A1–A20, L1–L15, F1–F16, M1–M12, P1–P16, B1–B6 לפי מסמך המתכונים.
6. **וואטסאפ:** חיבור הערוץ לסוכן 44092 (נחי, מהנייד); הודעות אוטומטיות ללקוחות = אפליקציית טיימליינס-איי-איי בתשלום (~25–40$ למשתמש לחודש) — החלטת המשך.
7. **פרויקט "סבג מאנדיי" ב-claude.ai לנייד** — נחי מדביק את הפרומפט מהקובץ ומפעיל את מחבר המאנדיי.
8. **עדכון הפרומפט המאסטר** לשינויים מהסשן: עדיפות=רווחיות (לא ווי-איי-פי), אין חיבור אנשי קשר↔תיקים, "תמיד תאריך".

## 4. החלטות מרכזיות שהתקבלו (של נחי)

1. לא משתמשים בסוכני ה-AI של מאנדיי לעבודה — קלוד קוד הוא הסוכן (כפל תשלום). 44092 = שער וואטסאפ בלבד.
2. **תיק = לקוח.** תיק עם מחלקה אחת = לקוח רגיל; כמה מחלקות = רב-מחלקתי (מבנה בלבד).
3. **עדיפות נמדדת אך ורק לפי תשלום/רווחיות הלקוח** — לא לפי מספר מחלקות.
4. **אין חיבור לוחות בין אנשי קשר לתיקי לקוחות** — העתקה באוטומציה + הסבר + סטטוס + לינק בלבד.
5. חברה = תיק נפרד לחלוטין לכל חברה; איש קשר אחד יכול להתחבר לכמה תיקים.
6. לקוח פרויקט (כמו זלמן ארן 1) מקבל גם מחלקת כספים ומחלקת ביצוע — שלא קיימות ללקוח רגיל.
7. **פקודה קפואה: "תמיד תאריך"** — אין פריט בלי תאריך, אין לוח בלי עמודת תאריך.
8. הפנימי נעול ופתוח למורשים בלבד.
9. כתיבה לנחי: עברית נקייה בלי מונחים לועזיים באמצע משפט (בעיית כיווניות).

## 5. שאלות פתוחות

1. לאן מנותבת תווית "ביצוע / פרויקטים" בנתב?
2. מי איש המכירות ברירת-מחדל לשיוך לידים (בני מובשוביץ?)
3. תוויות לוח הלידים בפועל שונות מהמשפך שבפרומפט (אין "בניסיון קשר"/"תואם") — ליישר את הלוח או את הפרומפט?
4. הודעות וואטסאפ אוטומטיות — האם להתחיל ניסיון חינם בטיימליינס-איי-איי?
5. מילוי עמודת "קישור לתיק" באיש הקשר בחזרה מהתיק — אוטומציה מובנית או משימת סוכן?
6. הועלה חשש אבטחה: נחי הדביק טוקן API של מאנדיי בצ'אט — הומלץ לחדש (Regenerate). *(הטוקן לא נכלל בתיעוד הזה בכוונה.)*
