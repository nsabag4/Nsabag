## 1. מטרת הפרויקט

חיבור וניהול של מערכת monday.com דרך Claude, ובהמשך: **עשיית סדר בחשבון** — מחיקת כפילויות ובנייה מחדש בצורה אחידה ומסודרת.

בקשות המשתמש (בסדר כרונולוגי):
1. "אני רוצה חיבור וניהול של מאנדיי" — חיבור למאנדיי וניהולו.
2. "אני רוצה שתעשה סדר תמחק כפילויות ותבנה את זה בצורה אחידה ומסודרת" — ניקוי וארגון מחדש של החשבון.
3. הבהרה: "יש כבר אחד אחד שרץ ברקע" + "שלא יהיה כפילות" — קיים תהליך/סוכן אחר שכבר רץ ברקע ומבצע את הסידור; אסור להריץ במקביל כדי שלא תיווצר כפילות.
4. שאלה: "למה חסרים לי שיחות קודמות?" — בירור על היעלמות שיחות קודמות מהממשק.
5. בקשה אחרונה: העלאת כל תוכן השיחה לריפו זה (המסמך הנוכחי).

## 5. החלטות שהתקבלו

1. **עצירת כל פעולה במאנדיי** — המשתמש הודיע שיש תהליך אחר שרץ ברקע ומבצע את הסידור; הוחלט לא להריץ במקביל כדי למנוע התנגשות וכפילות. ההחלטה: להמתין לסיום התהליך שברקע, ואז לבצע ביקורת חוזרת בלבד (להשלים רק את מה שחסר).
2. **מדיניות בטיחות שהוצעה (טרם אושרה סופית):** ארכוב במקום מחיקה לצמיתות — הפיך וניתן לשחזור. מחיקה לצמיתות רק לאשפה ודאית.
3. **עקרון עבודה דו-שלבי:** קודם ביקורת מלאה + הצגת תוכנית → רק אחרי אישור מפורש נוגעים במשהו.
4. **המבנה הממוספר החדש** (20 | משפטי, 30 | תכנון ובנייה, 40 | מימון, שנוצר ב-04/07/2026) זוהה כבסיס הנכון להתיישר אליו.

1. **מה בדיוק התהליך שרץ ברקע?** האם זה סשן Claude אחר, סוכן מאנדיי, או עבודה ידנית — והאם/מתי הוא מסיים? נדרש עדכון מהמשתמש כדי להריץ את הביקורת החוזרת.
