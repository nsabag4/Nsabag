נחי מבקש (08.07.2026): כשהוא כותב **סיבה והסבר** ולא רק פקודה — עליי (1) להוציא את הפקודה בפועל ולבצע אותה, ו-(2) **לשמור את הסיבות בנפרד, "קפואות"**, כדי שאם בעתיד ישאל "למה אמרתי לעשות 1-2-3" — אראה לו את הסיבה והוא ייזכר.

**Why:** נחי מקבל החלטות ומאבד עם הזמן את ההיגיון שמאחוריהן (ADHD); הסיבה הכתובה היא העוגן שמחזיר לו את ההקשר ומאפשר לו להצדיק "למה כך ולא אחרת". פקודה בלי סיבה = החלטה שהוא לא יוכל לשחזר.

**How to apply:** בכל הנחיה שיש בה נימוק — לתמצת את הפקודה לפעולה, ובמקביל לשמור את הסיבה: בזיכרון (כאן/בזיכרון הפרויקט), וגם כהערה קפואה ליד הקוד/ההגדרה שנוצרה (למשל תגובת `# Reason (frozen): ...`). כשמריצים פקודה קפואה של נחי — לצרף אליה תמיד את הסיבה. ראה [[sabag-monday-agent-system]] לדוגמאות (לימוד נסיעות, שיחות רכב).
