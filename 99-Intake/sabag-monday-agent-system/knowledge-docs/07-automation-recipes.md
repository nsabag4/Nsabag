# מתכוני אוטומציה — תרגום כל אוטומציה ממוספרת לבנייה במאנדיי — סבג

 _מדריך העבודה של הסוכן במצב הקמה/שיפור. לכל אוטומציה: הטריגר, התנאי והפעולה במונחי ה-Automations Center של מאנדיי._

## מקרא

-  🟢 **מובנה** — נבנה ישירות באוטומציות מאנדיי (Custom Automation).

-  🟡 **מובנה + עזר** — דורש עמודת נוסחה / עמודת עזר / כמה אוטומציות. עמודה חדשה = אישור נחי (חוק ברזל 6).

-  🔴 **סוכן** — לא ניתן באוטומציה מובנית; מבוצע על ידי הסוכן (בטריגר מתוזמן או באירוע), או מסומן לבדיקה בסריקה השבועית.

 **כלל זהב:** לפני בנייה — `get_board_info` לתוויות המדויקות. אחרי בנייה — בדיקת קצה-לקצה עם פריט דמה.

## מסלול הליד (A1–A20) — לוח הלידים

-  **A1** 🟢 When item created → assign בעלים (ברירת מחדל) + notify לבעלים.

-  **A2** 🟢 When item created → set תאריך פעולה הבאה = today.

-  **A3** 🟢 When item created → create item in לוח אנשי הקשר (+ connect). אם האינטגרציה של דף הנחיתה כבר יוצרת כרטיס — לא בונים כפול.

-  **A4** 🟢 When date (פעולה הבאה) arrives and status is "בניסיון קשר" → notify בעלים.

-  **A5** 🟢 When status is "בניסיון קשר" for 7 days → notify נחי.

-  **A6** 🟡 סיום מקצב 5 נגיעות → set status "מאגר עתידי". דורש עמודת "מס' נגיעה" (עדכון ידני של איש המכירות) + אוטומציה: when נגיעה = 5 and date passed → change status. חלופה: החלטת איש מכירות ידנית ביום 7.

-  **A7** 🟢 When status changes to "תואם" → set תאריך פעולה הבאה = 2 days from now.

-  **A8** 🟢 שתי אוטומציות: When status is "תואם" for 4 days → notify בעלים | for 7 days → notify נחי.

-  **A9** 🟢 When status changes to "פגישה נקבעה" and תאריך פגישה is empty → notify בעלים "חובה תאריך פגישה" + When תאריך פגישה is 1 day away → notify בעלים (כולל להזכיר ללקוח).

-  **A10** 🟢 When תאריך פגישה has passed and status is "פגישה נקבעה" → notify בעלים "לעדכן תוצאת פגישה".

-  **A11** 🟢 When status changes to "הצעה/מו"מ" → set תאריך מעקב = 3 days from now.

-  **A12** 🟢 When status is "הצעה/מו"מ" for 7 days → notify בעלים | for 14 days → notify נחי.

-  **A13** 🟡 When status changes to "נסגר" → create item in CRM. סיווג המסלול ופתיחת תיק במחלקה — 🔴 הסוכן, דרך פרוטוקול הקליטה.

-  **A14** 🟡 מונה המרות — עמודת נוסחה/ווידג'ט דשבורד שסופר "נסגר" השבוע. לא אוטומציה — דשבורד.

-  **A15** 🟡 מאנדיי לא חוסם מעבר סטטוס. קירוב: When status changes to "לא רלוונטי" and סיבה is empty → notify בעלים + נחי. השלמה: הסריקה השבועית 🔴 מדגלת "לא רלוונטי" בלי סיבה.

-  **A16** 🟢 When תאריך פעולה הבאה (כולל שעה) arrives and פעולה הבאה is "לחזור אליו" → notify בעלים.

-  **A17** 🟡 When רצינות changes to 4/5 and תאריך פעולה הבאה is empty → notify בעלים + נחי (אם תנאי empty לא נתמך — הסריקה השבועית 🔴 תופסת).

-  **A18** 🟢 When תאריך פעולה הבאה has passed and פעולה הבאה is "לחזור אליו" → notify בעלים.

-  **A19** 🟢 When פעולה הבאה changes to "לשלוח חומרים" → notify בעלים.

-  **A20** 🟢 When נשלחו חומרים (תאריך) changes → set פעולה הבאה = "פולו-אפ חומרים" + set תאריך פעולה הבאה = 2 days from now.

## משפטי (L1–L15) — הלוח המשפטי

-  **L1** 🔴 טעינת תבנית תתי-משימות לפי מסלול — הסוכן בפתיחת תיק (מאנדיי לא טוען תבנית תתי-משימות באוטומציה). טריגר עזר 🟢: When item created → notify הסוכן/אחראי "לטעון תבנית".

-  **L2** 🟢 When item created → create item במנהל המשימות: "הסכם שכ"ט + מכתב פתיחה — [שם התיק]", תאריך = today, אחראי = אחראי התיק.

-  **L3** 🟢 When תאריך פנימי is 2 days away → notify אחראי.

-  **L4** 🟢 When ממתינים-לצד-שני-עד has passed → create task "נדנוד" + notify אחראי. רישום יומן + הודעת לקוח — 🔴 הסוכן.

-  **L5** 🟢 ארבע אוטומציות: When תאריך נעול is 14/7/3/1 days away → notify אחראי (+נחי ב-3 וב-1).

-  **L6** 🟢 When תאריך דיון changes → create task "הכנה לדיון — [תיק]" עם תאריך = 7 ימים לפני. צ'קליסט ההכנה כתתי-משימות — 🔴 הסוכן.

-  **L7** 🔴 דיון שעבר בלי סיכום ביומן — לא ניתן לזהות Updates באוטומציה. הסוכן בודק בו-ביום (טריגר יומי) + הסריקה השבועית.

-  **L8** 🟢 When status changes to "חתימה" → set תאריך דיווח מיסוי = X days from now (נחי מגדיר X בתבנית) → מפעיל את L5.

-  **L9** 🟢 When שלב changes → notify אחראי "לעדכן את הלקוח". רישום היומן — 🔴 הסוכן.

-  **L10** 🟢 When שלב חיוב changes to "הושלם" → create task "להוציא חשבונית" (today).

-  **L11** 🟢 בלוח התזרים (סוכן הפנים): When תאריך תשלום passed by 1 day → status "ללחוץ עכשיו" | by 8 → "מתעכב" | by 31 → "אכיפה משפטית" + notify. פתיחת תיק גבייה במשפטי — 🔴 סוכן התפעול.

-  **L12** 🟡 עמודת "מס' סבב" + When מס' סבב changes to 4 → notify נחי + change status "ממתין להחלטת נחי".

-  **L13** 🟢 When status unchanged for 7 days (ליטיגציה: 3) → notify אחראי | 14 (ליטיגציה: 7) → נחי. תנאי "לא בהמתנה" — להחריג את סטטוס "ממתינים לצד שני" בתנאי האוטומציה.

-  **L14** 🟡 When status changes to "לסגירה" and סטטוס חיוב is not "שולם" → notify נחי + change status back. החלטה מתועדת — 🔴 ביומן.

-  **L15** 🔴 עדכון VIP יזום דו-שבועי — הסוכן בטריגר הדו-שבועי (או אינטגרציית אימייל בתבנית — שדרוג באישור).

## מימון (F1–F16) — לוח המימון

-  **F1** 🔴 טעינת תבנית + חישוב צפי — הסוכן. עמודת נוסחה לצפי 🟡 (אישור נחי).

-  **F2** 🟢 When item created → create task "הסכם שכ"ט + מכתב פתיחה" (today).

-  **F3** 🟡 נדנוד כל 3 ימים: When תאריך נדנוד arrives and מסמך status is not "התקבל" → notify + set תאריך נדנוד = 3 days from now (לולאה). אחרי 7 ימים → notify נחי (אוטומציה שנייה על תאריך פתיחת החוסר).

-  **F4** 🟢 When all subitems status = "התקבל" → change status to "הגשה" + create task "הגשה" (today).

-  **F5** 🟢 When status changes to "הוגש" → notify לקבוע "ממתינים עד"; When ממתינים-עד passed → create task נדנוד + notify.

-  **F6** 🟢 When תוקף אישור עקרוני is 21/14/7/3 days away → notify (4 אוטומציות). הסלמה כשהתיק לא מתקדם — 🔴 סריקה.

-  **F7** 🟢 When status changes to "שמאות" → create task "תיאום שמאי וגישה לנכס".

-  **F8** 🟢 When status changes to "שמאות נמוכה" (תווית ייעודית) → notify נחי. ההשוואה עצמה — 🔴 האחראי/הסוכן.

-  **F9** 🔴 יצירת צ'קליסט טרום-חתימה כתתי-משימות — הסוכן. טריגר עזר 🟢: When status changes to "אישור סופי" → notify.

-  **F10** 🟢 When מועד חתימה is 2 days away → notify אחראי. וידוא צ'קליסט ירוק — 🔴 הסוכן בו-ביום.

-  **F11** 🟢 When status changes to "חתימות בוצעו" → notify לקבוע "ממתינים להעברה עד"; When passed → notify נחי (אדום).

-  **F12** 🟡 עמודת נוסחה "מרווח (ימים)" = תאריך תשלום − צפי העמדה. אוטומציה על נוסחאות מוגבלת — הבדיקה הרציפה 🔴 אצל הסוכן (בכל נגיעה בתיק + סריקה שבועית).

-  **F13** 🟢 כמו L9. | **F14** 🟢 כמו L10. | **F15** 🟢 ספי 5/10 כמו L13. | **F16** 🔴 כמו L15.

## הצלבות (M1–M12) — משקיעים / נכסים / התאמות

-  **M1** 🟡 When item created (נכסים) → notify + set ברירות מחדל. תיוג "פרופיל חסר" על שדות ריקים — 🔴 סריקת הסוכן.

-  **M2** 🟢 When פתוח-להצעות changes to "כן" and הסכם תיווך is "חתום" → change status to "מוכן להתאמה" + notify הסוכן. ריצת ההתאמה — 🔴 הסוכן.

-  **M3** 🔴 ריצה הפוכה על משקיע חדש/מעודכן — הסוכן. טריגר עזר 🟢: When item created (משקיעים) → notify.

-  **M4** 🟢 When item created (לוח התאמות) → notify נחי.

-  **M5** 🟢 When status is "חדשה" for 3 days → notify אחראי | 7 → נחי.

-  **M6** 🟢 When status changes to "הוצגה" → set תאריך פולו-אפ = 3 days from now.

-  **M7** 🟢 When status changes to "לא מתאים" and סיבה is empty → notify. "פעם שלישית אותה סיבה" — 🔴 סריקת הסוכן.

-  **M8** 🔴 סגירת שאר ההתאמות של נכס שנמכר — בין-פריטי; הסוכן מבצע כשעסקה נסגרת. טריגר עזר 🟢: When status changes to "עסקה" → notify הסוכן.

-  **M9** 🟢 When תאריך רענון פרופיל passed by 60 days → create task "שיחת רענון".

-  **M10** 🟢 When all subitems (חסמים) done → change status to "נקי להצגה" + create task "לבחון עיתוי מכירה". ריצת התאמה מחדש — 🔴 הסוכן.

-  **M11** 🔴 נכס 14 יום בלי תנועה ובלי אף התאמה — תנאי בין-לוחי; הסריקה השבועית.

-  **M12** 🔴 קדימות VIP בהצגת התאמה — התנהגות הסוכן, לא אוטומציה.

## פרויקטים (P1–P16) — לוח הפרויקט

-  **P1** 🔴 טעינת מבנה + דוח אפס — הסוכן בפתיחת פרויקט.

-  **P2** 🟢 When אבן דרך status changes to "הושלמה בשטח" → create task "אישור מפקח".

-  **P3** 🟢 When status changes to "אושר ע"י מפקח" → create tasks "תשלום לקבלן" + "בקשת שחרור מהבנק". רישום בתזרים — 🔴 מסירה לסוכן הפנים.

-  **P4** 🟡 השוואת עלות בפועל מול מתוכנן — עמודת נוסחה + בדיקת הסוכן; חריגה → הסוכן יוצר פריט שינוי "מוצע".

-  **P5** 🟡 When תאריך אבן דרך is X days away → notify "לפתוח משימות הזמנה". יצירת משימות ההזמנה מהתבנית — 🔴 הסוכן.

-  **P6** 🟢 When תאריך הזמנה אחרון passed and status is not "הוזמן" → notify נחי + אחראי (דגל אדום).

-  **P7** 🟢 When status changes to "הוזמן" → notify לקבוע "ממתינים לאספקה עד"; When passed → notify. בחינת השפעה על אבני הדרך — 🔴 הסוכן.

-  **P8** 🟢 When מועד תשלום רוכש is 14 days away → create task "דרישת תשלום"; When passed → מדרגות הגבייה כמו L11.

-  **P9** 🔴 משוואת הכיסוי לחודשיים — חישוב רב-לוחי; הסוכן בסריקה השבועית + ווידג'ט דשבורד 🟡.

-  **P10** 🟡 עמודת נוסחה "% ניצול" + התראה מעל 85% אם נתמך; אחרת 🔴 סריקה.

-  **P11** 🟢 Every time period (לפי לו"ז הבנק) → create task "דיווח לבנק".

-  **P12** 🟢 When שינוי status changes to "מאושר" → notify + עדכון רווח דרך עמודת נוסחה 🟡 (סכום שינויים מאושרים). סף 1% — 🔴 הסוכן בודק לפני ביצוע.

-  **P13** 🟢 When פקיעת ביטוח is 30/14/7 days away → notify (3 אוטומציות).

-  **P14** 🟡 עלות עיכוב = ימי איחור × Burn Rate (נוסחה). יצירת פריט שינוי — 🔴 הסוכן.

-  **P15** 🟢 ספי 5/10 כמו L13.

-  **P16** 🔴 יומן פרויקט — הסוכן רושם Updates על כל אירוע מהותי.

## לוח הבוקר (B1–B6) — כל לוחות המחלקות

-  **B1** 🟢 When item created → set עדיפות "חשוב ולא דחוף" (אם ריק) + notify להשלמת סיווג.

-  **B2** 🟢 When תאריך נעול changes → set עדיפות "חשוב ודחוף".

-  **B3** 🟢 When VIP changes to "כן" → set עדיפות מדרגה למעלה.

-  **B4** 🟢 When תאריך ביצוע passed and status is not "בוצע" → set "באיחור" + notify אחראי. "ראש הרשימה מחר" — מובטח מהמיון בתצוגה (באיחור קודם).

-  **B5** 🟡 עמודת "מונה דחיות" (מספר). מאנדיי לא סופר לבד — האחראי/הסוכן מעדכן בכל דחייה; When מונה changes to 3 → notify נחי. פרוטוקול הפירוק — 🔴 הסוכן.

-  **B6** 🟢 Every time period (יומי 18:00) → notify סיכום; ריכוז פר מחלקה לדוח הבוקר — 🔴 הסוכן.

## סיכום כמותי

 🟢 מובנה ישיר: ~55 | 🟡 עם עמודת עזר/נוסחה (אישור נחי לעמודות): ~15 | 🔴 באחריות הסוכן (מתוזמן/אירוע/סריקה): ~19.

 **המשמעות:** רוב המערכת רצה לבד באוטומציות. מה שמסומן 🔴 הוא בדיוק מה שהסוכן מכסה בטריגרים המתוזמנים ובסריקה השבועית — זו חלוקת העבודה של מוסכמת האוטומציות בפרומפט המאסטר.

## דיוקים מהאימות (06.07.2026) — קרא לפני בנייה

1.  **B3 — סיווג מתוקן ל-🟡:** "קפיצת מדרגה" מותנית בערך הנוכחי אינה פעולה מובנית אחת — בונים אוטומציה נפרדת לכל מדרגה (חשוב-ולא-דחוף→חשוב-ודחוף וכו'), או מקבעים ל"חשוב ודחוף".

2.  **M2 — תנאי שלישי:** הטריגר המלא כולל גם "פרופיל נכס הושלם" (כל שדות החובה מלאים). באוטומציה המובנית בודקים פתוח-להצעות + הסכם תיווך; שלמות הפרופיל נאכפת דרך M1 וסריקת הסוכן — אין לסמן "מוכן להתאמה" ידנית על נכס עם "פרופיל חסר".

3.  **F13–F16:** ההפניות "כמו L9/L10/L13/L15" נכונות מבנית, אך בעת הבנייה חובה לתרגם לתוויות ולעמודות של לוח המימון בפועל (`get_board_info`) — לא להעתיק שמות מהלוח המשפטי.

4.  **L5 — נאמן למקור:** ההתראות המדורגות 14/7/3/1 הן לאחראי בלבד. הוספת נחי להתראות הקרובות היא אופציה — רק באישורו.

5.  **P8 — חישוב פיגורים:** חישוב הפיגורים לפי ההסכם (ריבית/הצמדה) הוא 🔴 באחריות הסוכן/האחראי — מדרגות הגבייה באוטומציה מסמנות את הסטטוס בלבד.
