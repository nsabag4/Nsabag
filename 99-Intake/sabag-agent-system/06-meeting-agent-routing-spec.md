# מפרט הניתוב המלא של הסוכן המסכם (כלשון נחי)

כשמגיעה הקלטת שיחה → הסוכן מסכם ב-2-3 נקודות, ואז מנתב לפי מה שהיה בשיחה.

## הכלל העל — עיקרון בית החולים
> "חולה שמגיע לבית חולים אחרי שעבר במיון ונקבע לו תאריך לניתוח אין לו צורך לעבור במיון שוב — כבר יודעים מה הסיפור שלו, ממה הוא סובל, מה כואב לו. אז כאן אותו דבר: לקוח שלנו שכבר יש לו תיק אני לא צריך להעביר אותו במיון, אני מעביר אותו ישר לתיק, ובתיק נפתחת משימה לטיפול בלקוח."

**אין תיק → נשאר בחדר המיון. יש תיק → עובר ללקוח (לתיק).**

## א. הסיכום עצמו
- יש את כל הפרטים **ויש תיק לקוח** → ישירות **לתיק הלקוח שלו**, ובתיק נפתחת משימה לטיפול.
- **אין תיק** → לחדר המיון התפעולי במאנדיי ("מאנדיי וורק"; נחי ציין 5094204623 — לוח לאישור).

## ב. התחייבות לחזור (החזרה לשיחה טלפונית)
- "אני אתקשר", "אחזור אליך", וכל דבר מהסוג הזה (זיכרון קצר-טווח) → **Any.do** — נודניק להתקשר בחזרה, **כולל שם וטלפון**.
- זה **לא** מנהל זמנים (מנהל זמנים = מונע-זמינות; החזרה = מונע-הבטחה).

## ג. פגישה שנקבעה בשיחה
- "ניפגש", "נקבע ל..." → **אירוע ביומן גוגל**.
  - פגישת חוץ → יומן "חוץ משרד".
  - פגישת משרד → יומן המשרד ("לו"ז משרד").
- הבחנה: פגישה = אירוע יומן (זמן+מקום). החזרה טלפונית = תזכורת Any.do (קצר-טווח).

## ד. התחייבות כספית
- "אני אשלם לך", "אני העברתי לך כסף", "אני צריך לקבל כסף" וכדומה — כל מילון הכסף/תמורה/התחייבויות (כבר קיים ב-prompt.md של הסוכן, רשת ד').
- → **בוט פיננסי** → שער הקלט והנתב הפיננסי (5094858663).
- **אבל** אם יש פרטים מלאים ללקוח ויש לו תיק → ישירות למאנדיי (לתיק).

## שני חדרי מיון
- **פיננסי** = שער הקלט והנתב 5094858663 (להתחייבויות כספיות).
- **תפעולי** = "מאנדיי וורק" (לסיכומים תפעוליים בלי תיק).

## חוקי-על שחלים על הכל
- **אין כפילות** — לפני כל יצירה, בדוק שאין דומה. קיים → השלם, אל תיצור כפול. (נחי עלול להעלות כפול כי שכח; סיכום שיחה עלול לחזור על משימה קיימת.)
- **אין תיק → מיון; יש תיק → לתיק** (עיקרון בית החולים).

## מצב מימוש (נכון לסוף הסשן)
- ✅ ROUTE_BY_CASE — לקוח עם תיק → סיכום נצמד לתיק (נבנה, נבדק).
- ✅ מילון הכסף + finance_mirror ללוח הפיננסי (נבנה).
- ✅ סגירת מעגל + דדופ (נבנה).
- ⏳ פגישה→יומן, החזרה→Any.do — דורש הרשאת כתיבה ליומן (הצעד הבא).
- ⏳ חדר מיון תפעולי + מנגנון "משימה בתיק" — דורש החלטות נחי.
