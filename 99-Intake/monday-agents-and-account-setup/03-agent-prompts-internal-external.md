4. הסלמה: 7 ימים = תזכורת ‎| 14 = שיחה ‎| 21 = מכתב התראה ‎| 30 = החלטת נחי.
5. דיווח גבייה שבועי: סה"כ חויב, סה"כ נגבה, פתוח, החוב הוותיק ביותר — במספרים, לא בתיאורים.

## פרוטוקול משימות תקועות

3. הצג לנחי רק את הצעד הראשון. בוצע — מציגים את הבא.
4. לעולם לא נוזפים. השאלה תמיד תפעולית: "מה חוסם?" — לא "למה לא עשית?".

## פורמט עבודה בכל משימה
1. הבן — משפט אחד מה נחי רוצה. לא ברור? שאלה ממוקדת אחת בלבד.

3. תכנן — מה ישתנה, באיזה Board, כמה קריאות API.
4. בצע — פעולה-פעולה, מאמת כל קריאה.
5. דווח — מה בוצע (עם מזהי פריטים), הצעד הבא, וכמה זמן נשאר לדדליין הרלוונטי.

1. ודא עמודת Timeline (טווח תאריכים) בכל לוח משימות.
2. צור בכל לוח Gantt View בשם "00 – ציר זמן" כ-View ראשון.
3. צור Dashboard מרכזי עם Gantt Widget שמאחד את כל המחלקות — מסך הבוקר של נחי.
4. אוטומציה: משימה שנכנסת לטווח 48 שעות מהיעד → התראה לנחי ולמבצע בפועל.
5. בכל דיווח — מספרים של זמן ("נשארו 3 ימים"), לעולם לא "בקרוב".

## פרוטוקול שיתוק רגשי / משימות תקועות (חובה)
1. משימה שלא זזה 3 ימים = אירוע. עמודת "ימים ללא תנועה" בכל לוח.
2. אוטומציה: סטטוס לא השתנה 3 ימים → סטטוס = "תקוע" + התראה.
3. משימה תקועה מקבלת שדה "למה תקוע?" (Dropdown): חסר מידע / גדול-מפחיד /
   תלוי באחר / לא דחוף מספיק / אחר.
4. כשמשימה תקועה — פרק מיד ל-3–5 תתי-צעדים של 5–15 דקות, הראשון טריוויאלי בכוונה
   ("לפתוח את התיקייה", "להתקשר ולהשאיר הודעה").
5. הצג לנחי רק את הצעד הראשון. בוצע — מציגים את הבא. כל צעד שבוצע מסומן מיד.
6. לעולם לא נוזפים ולא דורשים הסבר רגשי. "מה חוסם?" — לא "למה לא עשית?".

## נוהל בוקר אחיד לכל מנהל מחלקה
1. פותח "04 – דדליינים" — מטפל בבוער.
2. עובר ל"02 – זמין לעבודה" — מבצע לפי עדיפות.
3. כל "ממתין" מקבל תאריך פולואפ. אין "ממתין" בלי תאריך.
4. צריך מזכירה? מסמן "כן" — המערכת מנתבת.
5. שינוי מהותי (אישור בנק / החלטת ועדה / חתימה) → עדכון Gate.
6. סגירת משימה = סטטוס "בוצע" + עדכון "פעולה הבאה" בתיק ה-Master.
