# Intake: Timekeeper Agent Role Definition (שומר זמן / מנהל זמנים)

## מטרת הפרויקט (מה נעשה בשיחה הזו)

בשיחה הזו נחי סיכם וחידד את תפקיד הסוכן הרביעי במערכת סוכני סבג נדל"ן — **שומר זמן** (מנוהל גם כ-"מנהל זמנים", לשעבר "תזכורן"). המטרה: לצמצם את הגדרת התפקיד שלו לשני תפקידים בלבד ולחדד את הגבול בינו לבין הנודניק (Any.do), כדי שלא יטושטש עם סוכן המסכם (סיכומי שיחות) או עם מנגנון ההבטחות.

הסוכן הזה הוא **מנוע זמינות**: הוא היחיד שיודע בזמן אמת איפה נחי (GPS/Wi-Fi), מה עומסי הדרכים (TomTom), ומה רשום ביומן גוגל — ומזה הוא מפיק שתי פעולות בלבד: מתי לצאת לפגישה, ומתי יש זמן פנוי להתקשר ללקוחות.

## מה הוקם בפועל בשיחה הזו

1. עודכן `C:\dev\tizkoran\CLAUDE.md` (המפרט הטכני של שירות tizkoran) — נוספה בראש הקובץ סקציה "תפקיד הסוכן — הגדרה חדה" עם שני התפקידים וההבחנה מהנודניק.
2. נחי בעצמו כתב ושמר את **מסמך התפקיד הרשמי** בקובץ `C:\dev\agent-gate\prompts\timekeeper.md` — זהו הפרומפט המחייב שהסוכן קורא בתחילת כל עבודה (זה הקובץ שה-bot של agent-gate טוען עבור שיחות עם "שומר זמן").
3. עודכן `C:\Users\סבגנדלן\CLAUDE.md` (מרכזיית הניתוב הראשית של כל הסוכנים) — שורת הטבלה של סוכן 4 עודכנה כך שהיא מצביעה קודם כל על `agent-gate\prompts\timekeeper.md` כמסמך התפקיד הרשמי (ולא רק על ה-CLAUDE.md הטכני של tizkoran).
4. עודכן קובץ הזיכרון האוטומטי `sabag-monday-agent-system.md` עם נקודת המשך חדשה שמתעדת שהפרומפט נכתב ונשמר.

## מה תוכנן וטרם הוקם

- **אותו תהליך (מסמך תפקיד חד, קובץ נפרד ב-`agent-gate\prompts\`) לשלושת הסוכנים האחרים** — סבג מאנדיי (סוכן התפעול), הסוכן הפיננסי, וסוכן המסכם. נחי הציע זאת בסוף השיחה כהצעה לשיחה הבאה ("אם תרצה, בשיחה החדשה נעשה אותו דבר לשאר הסוכנים"). זה **לא בוצע** בשיחה הזו — רק לתפקיד שומר זמן.
- כל שאר הסעיפים הפתוחים שנרשמו קודם ב-project memory (ראה `04-project-state-memory.md`) לא טופלו בשיחה הזו — הם שייכים לשיחות קודמות ומובאים כאן רק להקשר.

- `01-session-log.md` — תמליל מלא ומילולי של השיחה (שני הסבבים): הודעות נחי, פעולות שבוצעו, ותוכן שנכתב.
- `02-prompts/timekeeper-VERBATIM.md` — הפרומפט המלא, מילה במילה, כפי שנחי שמר אותו בקובץ הרשמי.
- `03-source-docs/` — התוכן המלא (מילה במילה) של הקבצים שעודכנו: `tizkoran-CLAUDE.md`, מרכזיית הניתוב הראשית `CLAUDE.md`.
- `04-project-state-memory.md` — התוכן המלא של קובץ הזיכרון הרלוונטי (`sabag-monday-agent-system.md`), כרקע/הקשר בלבד — לא נוצר או שונה בשיחה הזו מלבד השורה החדשה בראש.
- `05-decisions-and-open-questions.md` — החלטות שהתקבלו ושאלות/הצעות פתוחות.
