# Sabag360 Framework — סבג נדל"ן

מקור האמת של מערכת הניהול והסוכנים של סבג נדל"ן. כל הידע חי בקבצים — הפרויקט זוכר, לא ה-AI.

## מבנה

| תיקייה | מה בפנים |
|---|---|
| `00-Constitution/` | החוקה: עקרונות-על מאושרים (עצמאות, שכבות זיכרון) + שאלות פתוחות |
| `04-Monday-Architecture/` | **Monday-Setup** (מה הוקם: לוחות, אוטומציות, סוכני מאנדיי) · **Monday-Operations** (הניהול השוטף) |
| `05-AI-Framework/` | **Call-Recording** — צינור הקלטות השיחות: טלפון ← תמלול ← סיכום ← ניתוב למאנדיי |
| `06-Development/` | **Smart-Time** — מנוע הזמינות (tizkoran): התראות יציאה לפי מיקום, עומסים, שיחות רכב |
| `07-Department-Agents/` | שמונה סוכני קלוד: מכירות, שיווק, משפטי, מימון, תכנון ובנייה, פרויקטים, פיננסי, והמנכ"ל שמעליהם |
| `08-עוזר-משפטי/` | עוזר משפטי עצמאי לטפסים והסכמים: שטרי מכר, הערות אזהרה, בדיקת מסמכים והכנה לחתימת עו"ד — נפרד לחלוטין מהמאנדיי ומסוכני המחלקות |
| `99-Intake/` | ארכיון חומרי הגלם מכל סשני הבנייה (07–11.07.2026) — קריאה בלבד |

## חוקי היסוד

1. **כל פרויקט וכל סוכן — תיקייה משלו, זיכרון משלו, חיים משלו.** קריסה של אחד לא נוגעת לשאר; סנכרון דרך המאנדיי בלבד.
2. **הזיכרון במחשב מחזיק את התוכן עצמו** — לא הפניות. המערכת עובדת מקומית גם בלי ענן; גיטהאב הוא הגיבוי והסנכרון.
3. **פותחים Claude Code בתוך תיקייה = מדברים עם הסוכן שלה.** ה-`CLAUDE.md` המקומי טוען זהות, גבולות וידע — בלי זליגה בין תיקיות.
4. טיוטה ← אישור נחי ← כתיבה. כל תוכן בעברית; שמות קבצים באנגלית.
