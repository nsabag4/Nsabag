# תזכורן (Tizkoran)

סוכן אישי שמונע איחורים ומנצל זמנים מתים.
הוא מצליב יומן גוגל + מיקום ובלוטות' מהטלפון + משימות ממאנדיי + עומסי תנועה,
ושולח הודעת טלגרם בדיוק ברגע הנכון.

## איך זה עובד (3 שכבות)
1. הטלפון = חיישנים. אפליקציית MacroDroid מזהה: התחברות לבלוטות' של הרכב, חיבור לרשת Wi-Fi מוכרת, כניסה לאזור GPS.
2. השרת = המוח. קוד Python שרץ במחשב שלך, מצליב יומן + מאנדיי + עומסים ומחליט.
3. טלגרם = הפה. ההודעות מגיעות אליך לטלפון.

## מה מקבלים
1. תזכורת יציאה: "לצאת בעוד 12 דקות" ואז "לצאת עכשיו!" – לפי נסיעה בעומסים + חניה + הליכה.
2. מצב רכב: התחברת לבלוטות' → "יש לך 60 דק' נסיעה, אפשר להספיק 3 שיחות: 1... 2... 3..." (מהלוח במאנדיי).
3. אזורים: נכנסת לאזור מוגדר → "אתה ליד X, יש לך שם: ...".

## שלב 1 – התקנת השרת במחשב (5 דקות)
1. העתק את התיקייה אל: `C:\dev\tizkoran`
2. פתח טרמינל בתוך התיקייה.
3. הרץ: `pip install -r requirements.txt`
4. העתק את הקובץ `.env.example` לקובץ חדש בשם `.env`
5. את מילוי הערכים בקובץ עושים בשלבים הבאים, אחד-אחד.

## שלב 2 – בוט טלגרם (5 דקות)
1. פתח בטלגרם את: @BotFather
2. שלח לו: `/newbot`
3. תן שם לבוט, למשל: Tizkoran
4. קיבלת טוקן. הדבק אותו ב-`TELEGRAM_BOT_TOKEN`
5. שלח הודעה כלשהי לבוט החדש שלך (למשל: היי).
6. פתח בדפדפן: `https://api.telegram.org/bot<הטוקן-שלך>/getUpdates`
7. חפש בתשובה: `"chat":{"id":123456789`
8. הדבק את המספר ב-`TELEGRAM_CHAT_ID`

## שלב 3 – חיבור היומן (3 דקות)
1. פתח את יומן גוגל במחשב.
2. גלגל שיניים → הגדרות.
3. בצד, תחת "הגדרות עבור היומנים שלי", בחר את היומן שלך.
4. גלול אל: "כתובת סודית בפורמט iCal".
5. העתק את הכתובת ל-`CALENDAR_ICS_URL`
6. חשוב מאוד: לכל פגישה ביומן חייבת להיות כתובת בשדה "מיקום". בלי כתובת אין חישוב נסיעה.

## שלב 4 – מאנדיי (5 דקות)
1. צור טוקן API חדש: תמונת פרופיל → Developers → My access tokens. (אל תשתמש בטוקן ישן שהופיע פעם בקבצים – הוא נחשב שרוף.)
2. הדבק ב-`MONDAY_API_TOKEN`
3. פתח את הלוח שמרכז חזרות ללקוחות. בכתובת הדפדפן מופיע `boards/123456789` – העתק את המספר ל-`MONDAY_CALLBACKS_BOARD_ID`
4. הרץ: `python print_monday_columns.py`
5. תקבל רשימת עמודות עם id לכל אחת.
6. העתק את ה-id של עמודת הסטטוס ל-`MONDAY_STATUS_COLUMN_ID`
7. העתק את ה-id של עמודת הטלפון ל-`MONDAY_PHONE_COLUMN_ID`
8. ב-`PENDING_STATUS_TEXTS` כתוב את שמות הסטטוסים שנחשבים "ממתין לחזרה", מופרדים בפסיק, בדיוק כמו שכתוב בלוח.

## שלב 5 – עומסי תנועה אמיתיים (רשות, 10 דקות)
בלי השלב הזה המערכת עובדת עם הערכת נסיעה קבועה (`DEFAULT_TRAVEL_MIN`).
1. היכנס אל: console.cloud.google.com
2. צור פרויקט חדש.
3. חפש והפעל את: Routes API (נדרש לחבר אמצעי תשלום, אך יש מכסה חינמית חודשית שמספיקה לשימוש אישי – הקוד חוסך בקריאות בכוונה).
4. צור מפתח: APIs & Services → Credentials → Create credentials → API key
5. הדבק ב-`GOOGLE_MAPS_API_KEY`

## שלב 6 – גשר בין הטלפון למחשב (Tailscale, 5 דקות)
כדי שהטלפון יגיע לשרת גם כשאתה בחוץ, על רשת סלולרית:
1. התקן Tailscale במחשב: tailscale.com/download
2. התקן Tailscale בטלפון מחנות האפליקציות.
3. התחבר בשניהם עם אותו חשבון.
4. במחשב, פתח את Tailscale והעתק את הכתובת שמתחילה ב-100 (למשל: 100.101.102.103).
5. זו כתובת השרת שלך מכל מקום. נשתמש בה בשלב 9.

## שלב 7 – פתיחת חומת האש במחשב (דקה)
1. פתח PowerShell כמנהל (קליק ימני → Run as administrator).
2. הרץ שורה אחת:
```
netsh advfirewall firewall add rule name="Tizkoran" dir=in action=allow protocol=TCP localport=8787
```

## שלב 8 – הפעלת השרת
1. מלא ב-`.env` גם `WEBHOOK_SECRET` – המצא מחרוזת ארוכה ואקראית (זו הסיסמה בין הטלפון לשרת).
2. לחץ פעמיים על `run.bat`
3. השאר את החלון פתוח. זה השרת.
4. בטל שינה למחשב: הגדרות → מערכת → צריכת חשמל → שינה: אף פעם (כשמחובר לחשמל).

## שלב 9 – MacroDroid בטלפון (15 דקות)
1. התקן MacroDroid מחנות האפליקציות (הגרסה החינמית מספיקה להתחלה).
2. תן הרשאת מיקום: "אפשר תמיד".
3. בטל עבורו אופטימיזציית סוללה (האפליקציה מציעה זאת בהגדרות).

### מאקרו א – בדיקת חיבור
1. לחץ: Add Macro
2. Trigger: בחר משהו ידני, למשל Applications → Shortcut Launched (או Button Widget).
3. Action: Connectivity → HTTP Request
4. Method: `POST`
5. URL: `http://<כתובת-Tailscale>:8787/event/test`
6. Content Type: `application/json`
7. Request Body:
```
{"secret":"הסוד-מהקובץ-env","action":"test"}
```
8. שמור והפעל. אמורה להגיע הודעת טלגרם: "תזכורן מחובר ועובד".

### מאקרו ב – כניסה לרכב
1. Trigger: Connectivity → Bluetooth Event → Device Connected → בחר את הרכב שלך.
2. Action: HTTP Request (POST, כמו קודם), אבל:
3. URL: `http://<כתובת-Tailscale>:8787/event/car`
4. Body:
```
{"secret":"הסוד","action":"connected"}
```

### מאקרו ג – יציאה מהרכב
1. אותו דבר, עם Trigger של Device Disconnected.
2. Body:
```
{"secret":"הסוד","action":"disconnected"}
```

### מאקרו ד – אזור (מאקרו אחד לכל אזור)
1. Trigger: Location → Geofence Trigger → Area Entered → סמן עיגול על המפה.
2. URL: `http://<כתובת-Tailscale>:8787/event/zone`
3. Body (שים לב ל-zone_id):
```
{"secret":"הסוד","zone_id":"haifa_industrial","action":"enter"}
```
4. חשוב: ה-zone_id חייב להיות זהה בדיוק למה שמוגדר בקובץ `zones.yaml`.

### מאקרו ה – Wi-Fi (מאקרו אחד לכל הרשתות!)
זיהוי לפי Wi-Fi מהיר יותר מ-GPS, לא צורך סוללה, ועובד גם בתוך מבנים.
1. Trigger: Connectivity → Wifi State Change → Connected to network → בחר: Any Network.
2. Action: HTTP Request (POST).
3. URL: `http://<כתובת-Tailscale>:8787/event/wifi`
4. Body:
```
{"secret":"הסוד","ssid":"{ssid}","action":"connected"}
```
5. את `{ssid}` לא כותבים ידנית: שלוש נקודות בשדה ה-Body → Magic Text → Wifi SSID.
6. זהו. השרת מזהה לבד לאיזה אזור שייכת הרשת לפי `zones.yaml`. רשת לא מוכרת פשוט מתעלמים ממנה.
7. הוספת מקום חדש בעתיד = שורה אחת ב-zones.yaml, בלי לגעת בטלפון.

### מאקרו ו – ניתוק Wi-Fi (רשות)
מדווח לשרת שעזבת את המקום (GPS ממילא יתפוס את זה, אז זה רק ליתר דיוק).
1. Trigger: Wifi State Change → Disconnected from network → בחר את הרשת הספציפית (למשל של המשרד).
2. URL: אותו `/event/wifi`
3. Body (כאן כותבים את שם הרשת ידנית, כי אחרי ניתוק המשתנה ריק):
```
{"secret":"הסוד","ssid":"Sabag_Office","action":"disconnected"}
```

### טיפ – שליחת מיקום מדויק
בכל Body אפשר להוסיף מיקום, וזה משפר את חישוב הנסיעה:
```
{"secret":"הסוד","action":"connected","lat":XX,"lon":YY}
```
במקום XX ו-YY: לחץ על שלוש הנקודות בשדה ה-Body → Magic Text → קטגוריית מיקום.

## שלב 10 – הגדרת האזורים שלך
1. פתח את `zones.yaml`
2. עדכן כתובת ונ"צ לכל אזור (נ"צ: לחיצה ארוכה על נקודה בגוגל מפות).
3. מלא `wifi_ssids`: שם הרשת בדיוק כמו שמופיע בהגדרות ה-Wi-Fi בטלפון (הבית, המשרד, וכל מקום עם רשת קבועה).
4. הוסף אזורים חדשים לפי הצורך.
5. אזור עם Wi-Fi – מכוסה אוטומטית על ידי מאקרו ה. אזור בלי Wi-Fi (רחוב, אזור תעשייה) – צור לו מאקרו GPS (מאקרו ד).

## בדיקה מלאה
1. הפעל את מאקרו הבדיקה → הודעה בטלגרם.
2. קבע פגישת ניסיון ביומן בעוד שעה, עם כתובת אמיתית בשדה מיקום.
3. כשיגיע הזמן – תגיע התראת יציאה, ו-15 דקות לפני כן תזכורת מוקדמת.
4. התחבר לבלוטות' של הרכב → תגיע הודעת שיחות.

## פתרון תקלות
1. אין הודעת טלגרם? בדוק טוקן ו-chat_id, ושחלון השרת רץ בלי שגיאות.
2. הטלפון לא מגיע לשרת? ודא ש-Tailscale דולק בשני המכשירים ושחוק חומת האש נוסף (שלב 7).
3. אין התראות יציאה? ודא שיש כתובת בשדה המיקום של הפגישה ביומן.
4. הנסיעה לא לפי עומסים? בדוק ש-Routes API מופעל ושהמפתח תקין. בינתיים המערכת משתמשת בהערכה קבועה.
5. אזור לא מזוהה? ודא שה-zone_id במאקרו זהה בדיוק לזה שבקובץ zones.yaml, ושל-MacroDroid יש הרשאת מיקום "תמיד".
6. חיבור Wi-Fi לא מזוהה? ודא ששם הרשת ב-zones.yaml זהה בדיוק לשם בטלפון, ושנתת ל-MacroDroid הרשאת מיקום (אנדרואיד דורש אותה כדי לקרוא שם רשת).

## מפת קבצים

3. `calendar_sync.py` – קריאת היומן.
4. `travel.py` – זמן נסיעה עם עומסים.
5. `monday_client.py` – שליפת לקוחות ומשימות ממאנדיי.
6. `notify.py` – שליחת טלגרם.
7. `zones.yaml` – האזורים שלך.
8. `state.py` – זיכרון קטן שמונע הודעות כפולות.
9. `print_monday_columns.py` – עוזר למצוא מזהי עמודות במאנדיי.
