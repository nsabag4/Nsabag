# שיתוף הסורק לכל המשרד 🌐

בברירת המחדל, אחרי ההתקנה הסורק זמין רק במחשב שאליו הוא מחובר ב-USB
(בכתובת `http://localhost:8090`). אם רוצים שכל מחשב במשרד יוכל לסרוק —
מריצים סקריפט אחד קטן על המחשב המארח, וזהו.

## מה עושים (פעם אחת, על המחשב שהסורק מחובר אליו)

1. ודאו שההתקנה הראשית (`install.ps1`) הסתיימה בהצלחה ושאפשר לסרוק
   מהדפדפן במחשב הזה.
2. פתחו PowerShell **כמנהל** (התחל ← הקלידו PowerShell ← קליק ימני ←
   "הפעל כמנהל").
3. עברו לתיקיית ההתקנה והריצו:

</div>

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
cd "<תיקיית-ההתקנה>\windows-bridge"
.\share-lan.ps1
```

<div dir="rtl">

4. הסקריפט ידפיס בסוף את הכתובות המדויקות לשאר המשרד, למשל:

</div>

```text
http://DESKTOP-OFFICE1:8090
http://192.168.1.42:8090
```

<div dir="rtl">

## איך סורקים ממחשב אחר במשרד

- **מהדפדפן**: גולשים לכתובת שהסקריפט הדפיס (למשל
  `http://DESKTOP-OFFICE1:8090`), בוחרים הגדרות ולוחצים Scan.
- **מ-NAPS2**: מוסיפים סורק eSCL עם Manual IP =
  שם-המחשב-המארח`:8090` (בדיוק כמו במדריך NAPS2, רק עם שם המחשב
  המארח במקום localhost).

## חשוב לדעת

- המחשב המארח (זה עם הסורק) צריך להיות **דלוק** כדי שאחרים יסרקו.
- השיתוף פתוח רק לרשתות בפרופיל Domain/Private של Windows — לא לרשתות
  ציבוריות.
- לממשק הסריקה **אין סיסמה** — הוא מיועד לרשת משרדית פנימית ואמינה
  בלבד.
- אם הסריקה מרחוק לא עובדת אבל מקומית כן: בדקו שחומת אש של צד שלישי
  (אנטי-וירוס) לא חוסמת את פורט 8090, ושהמחשבים באותה רשת.

## ביטול השיתוף

</div>

```powershell
.\share-lan.ps1 -Disable
```

<div dir="rtl">

זה מסיר את פתיחת הפורטים ואת חוקי חומת האש שהסקריפט הוסיף — ולא נוגע
בשום דבר אחר. הסריקה מהמחשב המארח ממשיכה לעבוד כרגיל.

</div>
