# מדריך התקנה מהיר — windows-bridge

מדריך זה מלווה אתכם, צעד אחר צעד, בהתקנת הפתרון המומלץ: הסורק Avision AV210C2 עובד על Windows 10/11 באמצעות הדרייבר הפתוח של SANE שרץ בתוך WSL2. בסוף התהליך תסרקו מהדפדפן או מתוכנת NAPS2.

ההתקנה נעשית **פעם אחת בלבד** (רצוי על ידי איש IT או משתמש עם הרשאות מנהל). אחריה — פשוט סורקים.

## לפני שמתחילים — דרישות קדם

1. **Windows 10 גרסה 2004 ומעלה, או Windows 11.** לבדיקה: הקישו על מקש Windows‏ + R, הקלידו `winver` ואשרו. אם הגרסה ישנה יותר — עדכנו את Windows קודם.
2. **חשבון עם הרשאות מנהל מערכת (Administrator)** — נדרש להתקנה בלבד.
3. **חיבור אינטרנט** — ההתקנה מורידה כמה מאות MB (אובונטו, חבילות, קוד מקור).
4. **הסורק מחובר ודלוק**:
   - כבל ה-USB מחובר **ישירות** ליציאה במחשב — לא דרך מפצל (Hub) ולא דרך תחנת עגינה. במחשב נייח עדיפה יציאה אחורית.
   - **ספק הכוח 24V של הסורק מחובר לחשמל** ונורית ההפעלה דולקת. סורק בלי ספק כוח לא יופיע בכלל ברשימת ההתקנים — זו התקלה הנפוצה ביותר.
5. כ-10 דקות זמינות (ועוד אתחול אחד של המחשב, אם WSL לא היה מותקן קודם).

## שלב 1: פתיחת PowerShell כמנהל מערכת

1. לחצו על תפריט התחל והקלידו `PowerShell`.
2. לחצו קליק ימני על התוצאה "Windows PowerShell" ובחרו באפשרות ההרצה כמנהל מערכת ("Run as administrator" / "הפעל כמנהל").
3. אשרו את חלון בקרת חשבון המשתמש (UAC) שנפתח.
4. בחלון הכחול שנפתח, עברו לתיקיית הפרויקט. לדוגמה, אם המאגר נמצא ב-<span dir="ltr">`C:\Tools\scanner`</span>:

</div>

```powershell
cd C:\Tools\scanner\windows-bridge
Set-ExecutionPolicy Bypass -Scope Process
```

<div dir="rtl">

הפקודה `Set-ExecutionPolicy Bypass -Scope Process` מתירה הרצת סקריפטים **בחלון הזה בלבד** — היא לא משנה שום הגדרה קבועה במחשב. אם תופיע שאלת אישור, הקלידו `Y` ולחצו Enter.

## שלב 2: הרצת סקריפט ההתקנה

</div>

```powershell
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

<div dir="rtl">

הסקריפט מדפיס 7 שלבים ממוספרים, כל אחד בעברית ובאנגלית. זה מה שתראו:

**שלב 1/7 — בדיקת גרסת Windows ווירטואליזציה.** אם הגרסה ישנה מדי, הסקריפט עוצר עם הסבר. אם מופיעה אזהרה על וירטואליזציה — ראו [פתרון תקלות](TROUBLESHOOTING.he.md).

**שלב 2/7 — התקנת WSL2 ואובונטו.** אם WSL לא היה מותקן, כאן תופיע ההודעה החשובה ביותר:

> IMPORTANT: A reboot is most likely required now. / חשוב: כנראה נדרשת הפעלה מחדש של המחשב.

במקרה כזה:

1. אתחלו את המחשב.
2. אם אחרי האתחול נפתח חלון של Ubuntu שמבקש שם משתמש וסיסמה — הקלידו שם משתמש (באותיות אנגליות קטנות) וסיסמה שתזכרו, וסגרו את החלון.
3. פתחו שוב PowerShell כמנהל (שלב 1 למעלה) והריצו שוב את `install.ps1`. הסקריפט מזהה מה כבר הותקן וממשיך הלאה — הרצה חוזרת תמיד בטוחה.

**שלב 3/7 — התקנת usbipd-win.** זהו הרכיב שמעביר את חיבור ה-USB של הסורק אל תוך WSL. מותקן אוטומטית דרך winget.

**שלב 4/7 — איתור ושיתוף הסורק.** הסקריפט מחפש את ההתקן `0638:0a3a`. פלט תקין נראה כך:

</div>

```
    OK: Scanner found at BUSID 1-3
    OK: Scanner is now shared with usbipd (persistent across reboots)
```

<div dir="rtl">

אם מופיעה שגיאה שהסורק לא נמצא — בדקו כבל, יציאת USB וספק 24V, והריצו שוב.

**שלב 5/7 — הגדרת אובונטו.** השלב הארוך ביותר (כמה דקות): התקנת SANE, הפעלת דרייבר ה-avision, כללי הרשאות ל-USB, וקומפילציה של שרת הסריקה AirSane. תראו הרבה שורות שמתחילות ב-<span dir="ltr">`[wsl]`</span> — זה תקין. אם השלב נכשל, ההודעה תפנה אתכם לשורת השגיאה המדויקת.

**שלב 6/7 — חיבור הסורק ל-WSL ורישום משימה מתוזמנת.** נרשמת משימה בשם `ScannerBridge-AttachAV210C2` שרצה בכל כניסה למחשב ומחברת את הסורק מחדש אוטומטית — גם אחרי אתחול וגם אחרי ניתוק וחיבור של הכבל.

**שלב 7/7 — אימות.** הסקריפט בודק שהדרייבר באמת רואה את הסורק. פלט תקין:

</div>

```
    OK: SANE sees the scanner: device 'avision:libusb:...' is a Avision AV210C2 ...
    OK: AirSane eSCL server is running on port 8090
```

<div dir="rtl">

ולבסוף:

</div>

```
=========================================================
 Installation finished! / ההתקנה הסתיימה!
=========================================================
```

<div dir="rtl">

## שלב 3: סריקה ראשונה מהדפדפן

1. שימו דף במזין הדפים של הסורק.
2. פתחו דפדפן (Edge / Chrome) וגלשו אל:
   - <span dir="ltr">`http://localhost:8090`</span> — ממשק AirSane הבסיסי, מאפשר סריקה ישירה לקובץ.
   - <span dir="ltr">`http://localhost:8080`</span> — ממשק scanservjs הנוח יותר (מותקן כברירת מחדל): בחירת רזולוציה, צבע/שחור-לבן, סריקת כמה עמודים ל-PDF אחד, והורדה של הקובץ.
3. לחצו על כפתור הסריקה בממשק — הסורק ימשוך את הדף והקובץ יופיע להורדה.

טיפ: שמרו את אחת הכתובות כסימנייה בדפדפן, או כקיצור דרך על שולחן העבודה.

## שלב 4 (מומלץ): הוספת הסורק ב-NAPS2

NAPS2 היא תוכנת סריקה חינמית ונוחה ל-Windows, מצוינת לסריקת מסמכים מרובי עמודים ל-PDF, כולל OCR בעברית.

1. הורידו והתקינו NAPS2 **בגרסה 7.5 ומעלה** (מומלץ הגרסה העדכנית): <span dir="ltr">https://www.naps2.com/download</span>
2. פתחו NAPS2, היכנסו לניהול פרופילי הסריקה (Profiles) וצרו פרופיל חדש.
3. בבחירת ההתקן בחרו בדרייבר **ESCL Driver**.
4. אל תחכו לחיפוש האוטומטי — בחרו באפשרות ההזנה הידנית של כתובת (**Manual IP**) והקלידו בדיוק:

</div>

```
localhost:8090
```

<div dir="rtl">

5. שמרו את הפרופיל ולחצו סרוק (Scan).

הערה: אם שדה הכתובת הידנית לא מקבל את הסיומת `:8090` בגרסת ה-NAPS2 שלכם — ראו את הפתרון ב-[NAPS2.he.md](NAPS2.he.md).

מדריך מלא כולל פרופיל מומלץ ו-OCR בעברית: [NAPS2.he.md](NAPS2.he.md)

## ומה עם אפליקציית "סריקה" המובנית של Windows?

ל-Windows 10/11 יש תמיכה מובנית בסורקי eSCL, אבל הוספת סורק דרך הגדרות Windows ("מדפסות וסורקים" ← הוספת התקן) מסתמכת על גילוי אוטומטי ברשת (mDNS). כשהסורק "משודר" מתוך WSL אל אותו מחשב עצמו, הגילוי הזה **לא אמין** — לפעמים יעבוד, לרוב לא, ואין ב-Windows דרך להוסיף סורק eSCL לפי כתובת ידנית.

לכן ההמלצה שלנו חד-משמעית: **דפדפן או NAPS2.** אם בכל זאת הסורק הופיע לכם ברשימת ההתקנים להוספה — אפשר להוסיף אותו ולסרוק מאפליקציית "סריקה" של Windows, אבל אל תבנו על זה: אם הוא נעלם, חזרו לדפדפן/NAPS2.

## הסרה

כדי להסיר את הפתרון ולהחזיר את הסורק ל-Windows (למשל לצורך VueScan), פתחו PowerShell כמנהל בתיקיית `windows-bridge` והריצו: <span dir="ltr">`powershell -ExecutionPolicy Bypass -File .\uninstall.ps1`</span>. להסרה מלאה כולל מחיקת אובונטו (מוחק את כל המידע שבתוכה!) הוסיפו את המתג <span dir="ltr">`-RemoveDistro`</span>.
