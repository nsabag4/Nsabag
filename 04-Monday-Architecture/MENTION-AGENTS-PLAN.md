## 2. הארכיטקטורה

```
נחי כותב עדכון על פריט בלוח: "@סוכן משפטי — מה הסטטוס?"
  ← אוטומציית מאנדיי: אירוע עדכון-חדש ← Send webhook (מובנה, חינם)
  ← שער הסוכנים, נתיב /monday/mention (הרחבה של /monday/webhook הקיים)
  ← אימות סוד + דדופ 60 שניות + זיהוי שהעדכון מתייג סוכן (ולא נכתב על-ידי סוכן — מניעת לולאה)
  ← ניתוב לפי board_id ← תיקיית הסוכן (טבלה בסעיף 3)
  ← claude -p בתיקיית הסוכן: החוקה + MEMORY.md + טקסט השאלה + הקשר הפריט
  ← התשובה נכתבת כעדכון-תגובה על הפריט, חתומה: "— ⚖️ סוכן משפטי (Claude), [תאריך שעה]"
  ← MEMORY.md מתעדכן (מה נשאל, מה נענה) — הרצף נשמר לכל הדלתות
```

## 4. כללי בטיחות (מחייבים)

1. **מניעת לולאות:** עדכון שנכתב על-ידי סוכן — לעולם לא מפעיל סוכן.
2. **דדופ:** אירוע כפול תוך 60 שניות = מטופל פעם אחת.
3. **בלם הצפה:** מקסימום 2 תשובות במקביל; מעבר — תור.
4. **כשל בשקט אסור:** תקלה ← התראת טלגרם לנחי + יומן.
5. **הסוכן עונה — לא מבצע פעולות מסוכנות מהאזכור:** מחיקה/שינוי מבנה/כספים — לעולם לא מטריגר אזכור (חוקים 5, 12).
6. תלות מוצהרת: המנוע רץ על המחשב במשרד (עד הגירת ענן) — מחשב כבוי = הסוכן לא עונה בלוח.
