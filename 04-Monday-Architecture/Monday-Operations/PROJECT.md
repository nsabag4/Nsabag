# PROJECT.md — Monday-Operations | הניהול השוטף של המאנדיי

> מקור האמת של פרויקט Monday-Operations (פרויקט 2 מתוך 4 במסגרת Sabag360).
> נכתב על בסיס חומרי הקליטה (99-Intake) מסשני 04–09.07.2026, באישור נחי (11.07.2026).

# Purpose

הניהול השוטף של חשבון המאנדיי של סבג נדל"ן: קליטת משימות ונתונים ללוחות הנכונים, שליפת נתונים בזמן אמת, מעקב אחר תנועת המשימות עד סגירתן, דוחות קבועים והתרעות — הכול על בסיס התשתית הקיימת, בלי לשנות אותה תוך כדי עבודה.

# Scope

**בפנים:**
- מצב "שוטף" (עבודת היום-יום) ומצב "שיפור" (אוטומציות ואינטגרציות על תשתית קיימת).
- שני מרחבי העבודה: חוץ (לקוחות, מחלקות, לידים) ופנים (מנכ"ל/CFO, תזרים, כסף).
- הטקסים הקבועים: דוח בוקר יומי, סריקה שבועית, בדיקות תקינות אינטגרציה.

**בחוץ:**
- הקמת תשתית — לוחות, עמודות, סביבות, אוטומציות חדשות — שייכת ל-Monday-Setup (מצב "הקמה"). כלל ההכרעה: פעולה חד-פעמית שבונה תשתית = הקמה; פעולה מחזורית שמתפעלת את הקיים = שוטף.
- הקלטת שיחות וסיכומן — פרויקט Call-Recording.
- ניהול זמן ומיקום — פרויקט Smart-Time.

# Responsibilities

- קליטת כל משימה נכנסת לפי פרוטוקול הקליטה (ראה Workflow).
- שליפת נתונים אך ורק מהמאנדיי בזמן אמת — לעולם לא מהזיכרון.
- מעקב תנועה: זיהוי תקועים, באיחור, יתומים — והסלמה לפי הספים.
- הפקת דוח הבוקר היומי והסריקה השבועית.
- בדיקת תקינות שוטפת של האינטגרציות (ליד נכנס ⇦ כרטיס באנשי קשר).
- דיווח לנחי בפורמט קבוע: מה בוצע ← קישור ← הצעד הבא.

# Inputs

- הוראות ונתונים "יבשים" מנחי (טקסט/קול, דרך הבוטים או ישירות).
- לידים נכנסים מדפי נחיתה ומערוצי הפרסום (דרך אינטגרציות מאנדיי).
- פריטים חדשים בלוחות הקליטה: חדר מיון Inbox ‏(5094204623), שער הקלט והנתב (5094858663).
- לוחות המאנדיי עצמם — מקור הנתונים היחיד לשליפות.

# Outputs

- פריטים מעודכנים בלוחות עם מזהה וקישור לכל פעולה.
- דוח בוקר יומי (עד 3 פריטים בקטגוריה, לפי כללי ADHD).
- דוח סריקה שבועית ממוין לפי דחיפות.
- התרעות: תקוע 7+ ימים ← אחראי; 14+ ימים ← נחי; VIP — ספים קצרים.
- הערות שיפור (פגמים שהתגלו בשוטף — מטופלים אחר-כך במצב שיפור).

# Rules

חוקי הברזל (המקור המלא: הפרומפט המאסטר, ראה AGENTS.md):

1. **אין אישור בלי ביצוע** — דיווח "עודכן" רק אחרי ביצוע בפועל עם מזהה פריט וקישור מה-API. נכשל = מדווחים שנכשל.
2. **שליפה מהמאנדיי בלבד** — נתון שלא נמצא ⇦ "לא נמצא בלוח". אסור להשלים מהזיכרון או לנחש.
3. **קריאת מבנה הלוח לפני כל כתיבה** (get_board_info: מזהי עמודות, סוגים, תוויות).
4. **עדכון פריט קיים רק אחרי התאמת שם מדויקת** — ספק = שאלה לנחי.
5. **אין יצירת לוחות/עמודות/קבוצות בלי אישור מפורש מנחי.**
6. **"תמיד תאריך"** — אין פריט בלי תאריך, אין לוח בלי עמודת תאריך (פקודה קפואה).
7. **עדיפות = רווחיות הלקוח בלבד** — לא מספר מחלקות, לא ותק.
8. **בשוטף לא נוגעים במבנה** — פגם שהתגלה נרשם כהערת שיפור וממשיכים.
9. **סדר עדיפויות בשיפורים:** אוטומציה מובנית ← אינטגרציה מובנית ← גורם חיצוני רק באישור נחי.
10. **כתיבה לנחי בעברית נקייה** — בלי מונחים לועזיים באמצע משפט (בעיית כיווניות).

# Workflow

**פרוטוקול קליטת משימה (בסדר הזה, תמיד):**
1. פנים או חוץ? משימת פנים ⇦ סוכן הפנים, לא ממשיכה בפרוטוקול.
2. זיהוי מחלקה: משפטי / מימון / תכנון ובניה / מכירות / ניהול פרויקטים.
3. קישור לכרטיס הלקוח; לקוח חדש ⇦ כרטיס CRM + כרטיס אנשי קשר + תיעוד מקור (ליד/פה-לאוזן). לקוח 360 (VIP) ⇦ קישור גם למטה הלקוח.
4. מילוי שדות חובה: שם, לקוח, אחראי, סטטוס, תאריך יעד, עדיפות.
5. חסר שדה קריטי ⇦ שאלה ממוקדת אחת או קליטה עם תיוג "חסר מידע" — לא עוצרים.
6. ביצוע ← אישור עם מזהה וקישור. משימה מורכבת ⇦ תתי-משימות.

**ניתוב חדר המיון:** עמודת "ניתוב אל" ⇦ אוטומציות יוצרות פריט בלוח המחלקה בקבוצת "לטיפול עכשיו" ומסמנות "בוצע ניתוב" (התשתית — ב-Monday-Setup).

**משפך לידים:** נכנס ← תואם ← פגישה ← מו"מ ← נסגר; כל ליד מועתק אוטומטית לאנשי קשר; מדידת יחס המרה ופילוח ערוצים שבועי.

**מסלולי המחלקות (התהליכים המלאים במסמכי ה-sabag-*):** משפטי — 4 מסלולים (חוזים / מכר / מו"מ מסחרי / ליטיגציה לפי מועדים דיוניים); מימון — ממקדמה עד העמדת כסף עם צפי סיום מהיום הראשון; תכנון ובניה; הצלבות משקיעים↔נכסים; ניהול פרויקטים.

**טקסים קבועים:** דוח בוקר (רוטינת sabag-morning-report, ‏~07:00) | סריקה שבועית (sabag-weekly-scan, ראשון 08:00) + תזכורת דפי בנק.
